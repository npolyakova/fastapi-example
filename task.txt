1. Склонируйте этот проект и запустите приложение:
    1. Установите зависимости pip install uvicorn fastapi
    2. Запустите код python main.py
    3. Перейдите на http://localhost:8002/(не 0.0.0.0:8002!),если видите message со строки 21 файла main.py, всё получилось
    4. Откройте Swagger - http://localhost:8002/docs

2. Измените API, согласно вашему порядковому номеру в списке группы:
    1. Магазин обуви - products замените на товары обувного магазина, сохраняется 3 атрибута (например: название, размер, цена). Добавьте операцию CREATE для добавления новых товаров.
    2. Список студентов - products замените на список студентов, количество атрибутов увеличивается до 5 (например: имя, группа, курс, средний балл, год рождения). Добавьте операцию DELETE.
    3. Автомобильный каталог - products замените на список автомобилей, количество атрибутов - 4 (марка, модель, год выпуска, цена). Добавьте операцию UPDATE.
    4. Книжный магазин - products замените на список книг, 4 атрибута (название, автор, жанр, цена). Добавьте операцию GET по ID.
    5. Рецепты - products замените на список рецептов, 5 атрибутов (название, категория, время приготовления, сложность, ингредиенты). Добавьте операцию поиска по категории.
    6. Фильмы - products замените на список фильмов, 4 атрибута (название, режиссер, год, рейтинг). Добавьте операцию сортировки по рейтингу.
    7. Сотрудники компании - products замените на список сотрудников, 5 атрибутов (ФИО, должность, отдел, зарплата, дата найма). Добавьте операцию фильтрации по отделу.
    8. Тренажерный зал - products замените на список абонементов, 4 атрибута (тип абонемента, цена, срок действия, количество занятий). Добавьте операцию PUT для обновления.
    9. Ресторанное меню - products замените на список блюд, 4 атрибута (название, категория, цена, вес). Добавьте операцию DELETE и GET по категории.
    10. Аптека - products замените на список лекарств, 5 атрибутов (название, производитель, цена, срок годности, наличие). Добавьте операцию поиска по производителю.
    11. Спортивный инвентарь - products замените на товары спорттоваров, 3 атрибута (название, тип спорта, цена). Добавьте операцию CREATE и фильтрация по типу спорта.
    12. Музыкальная библиотека - products замените на список альбомов, 4 атрибута (исполнитель, название альбома, год, жанр). Добавьте операцию DELETE и поиск по исполнителю.
    13. Зоомагазин - products замените на товары для животных, 4 атрибута (название, категория, цена, вид животного). Добавьте операцию фильтрации по виду животного.
    14. Косметика - products замените на список косметических средств, 4 атрибута (название, бренд, тип, цена). Добавьте операцию UPDATE цены.
    15. Электроника - products замените на список электроники, 4 атрибута (название, бренд, цена, характеристики). Добавьте операцию поиска по бренду.
    16. Мебельный магазин - products замените на список мебели, 4 атрибута (название, тип, материал, цена). Добавьте операцию фильтрации по типу.
    17. Игровая библиотека - products замените на список видеоигр, 5 атрибутов (название, жанр, платформа, год, рейтинг). Добавьте операцию сортировки по году.
    18. Цветочный магазин - products замените на список цветов, 4 атрибута (название, тип, цена, сезонность). Добавьте операцию поиска по сезону.
    19. Туристическое агентство - products замените на список туров, 5 атрибутов (страна, город, цена, длительность, отель). Добавьте операцию фильтрации по стране.
    20. Канцелярский магазин - products замените на товары для офиса, 4 атрибута (название, тип, цена, производитель). Добавьте операцию CREATE и поиск по производителю.

3. Для удобства выполнения CRUD операций, вынесите работу с данными в отдельный JSON файл.
4. Для GET запроса добавьте обработку параметра (parameter) sorting. Если его значение asc, сортируйте список от А до Я, если desc - наоборот.
5. Проведите тестирование своего функционала в Swagger. Отправьте запрос на POST/PUT/DELETE, а затем запрос на GET,
сделайте скриншоты обоих запросов, на них должно быть видно, что объект создан и отображается в списке объектов в результате гет-запроса.
6. Проведите тестирование с помощью инструмента POSTMAN. Отправьте GET-запрос с разными способами сортировки. Сделайте скриншоты обоих запросов.
7. Запушьте код в отдельную ветку гитхаб репозитория, который был создан для прошлого домашнего задания. Откройте пулл-реквест, вложите в него скриншоты из пункта 4 и 5.
8. В ответ на задание приложите ссылку на созданный пулл-реквест.
